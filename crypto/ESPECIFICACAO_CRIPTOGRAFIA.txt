================================================================================
ESPECIFICAÇÃO TÉCNICA DA CRIPTOGRAFIA
Projeto: Sistema de Monitoramento de Sensores - Raspberry Pi 3
================================================================================

1. ALGORITMO DE CRIPTOGRAFIA
   - Nome: AES-256 (Advanced Encryption Standard)
   - Implementação: Fernet (Python Cryptography Library)
   - Modo: CBC (Cipher Block Chaining) com HMAC para autenticação
   - Padrão: Criptografia simétrica autenticada

2. DERIVAÇÃO DE CHAVE (KDF)
   - Algoritmo: PBKDF2-HMAC (Password-Based Key Derivation Function 2)
   - Função Hash: SHA-256
   - Iterações: 100.000 (proteção contra ataques de força bruta)
   - Tamanho da Chave: 256 bits (32 bytes)
   - Salt: 128 bits (16 bytes) - gerado aleatoriamente

3. SEGURANÇA IMPLEMENTADA
   - Padrão Militar: AES-256 é usado por governos e instituições financeiras
   - Autenticação: HMAC garante integridade dos dados
   - Proteção contra Força Bruta: 100.000 iterações no PBKDF2
   - Senha Temporária: Validade de 20 segundos após geração
   - Salt Único: Proteção contra ataques de dicionário e rainbow tables

4. BIBLIOTECA UTILIZADA
   - Nome: cryptography
   - Versão Mínima: 3.0+
   - Linguagem: Python 3.6+
   - Instalação: pip3 install cryptography

5. CARACTERÍSTICAS DA SENHA TEMPORÁRIA
   - Tamanho: 24 caracteres
   - Composição: Letras maiúsculas, minúsculas, números e símbolos
   - Geração: Aleatória usando secrets (criptograficamente seguro)
   - Validade: 20 segundos
   - Uso Único: Cada criptografia gera nova senha

6. ARQUIVOS GERADOS
   - dados_criptografados_[timestamp].bin - Dados criptografados
   - salt.bin - Salt para derivação de chave
   - SENHA_PARA_GRUPO_[timestamp].txt - Senha e instruções

7. PROCESSO DE CRIPTOGRAFIA
   Passo 1: Coleta de dados dos sensores (30 segundos)
   Passo 2: Geração de senha temporária aleatória
   Passo 3: Derivação de chave usando PBKDF2-HMAC-SHA256
   Passo 4: Criptografia dos dados com AES-256 (Fernet)
   Passo 5: Salvamento do arquivo criptografado (.bin)
   Passo 6: Geração de arquivo com senha e instruções

8. PROCESSO DE DESCRIPTOGRAFIA
   Passo 1: Recebimento dos arquivos (.bin e salt.bin)
   Passo 2: Recebimento da senha temporária (válida por 20s)
   Passo 3: Derivação da mesma chave usando PBKDF2-HMAC-SHA256
   Passo 4: Descriptografia dos dados com AES-256 (Fernet)
   Passo 5: Conversão para JSON legível

9. NÍVEL DE SEGURANÇA
   - Resistência a Força Bruta: Praticamente impossível
   - Tempo Estimado para Quebrar: Milhões de anos
   - Proteção contra Interceptação: Alta (mesmo via HTTPS)
   - Proteção Temporal: Senha expira em 20 segundos

10. COMPATIBILIDADE
    - Sistemas Operacionais: Linux, Windows, macOS
    - Hardware: Raspberry Pi 3, computadores convencionais
    - Python: Versão 3.6 ou superior
    - Biblioteca: cryptography (disponível via pip)

================================================================================
CONCLUSÃO
================================================================================

O sistema implementa criptografia de nível militar (AES-256) com derivação
de chave robusta (PBKDF2-HMAC-SHA256 com 100.000 iterações) e senhas
temporárias de 20 segundos, garantindo segurança máxima na transmissão
dos dados coletados pelos sensores do Raspberry Pi.

A combinação de AES-256, PBKDF2, HMAC e senhas temporárias torna o sistema
praticamente impossível de ser quebrado sem a senha correta, mesmo com
ferramentas avançadas de criptoanálise.
